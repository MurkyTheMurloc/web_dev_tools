# Grill Me

Use this skill when the user wants a plan, architecture, or design to be
pressure-tested.

- Question assumptions directly.
- Walk through the decision tree branch by branch.
- Surface hidden dependencies, tradeoffs, and unanswered questions.
- Keep going until there is a shared understanding of the plan.
- If a question can be answered by exploring the codebase, inspect the code
  instead of asking the user.
