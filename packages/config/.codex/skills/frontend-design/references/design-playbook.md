# Frontend Design Playbook

Use these as starting directions when you do not already have a strong design
instinct.

## Pick One Of These Directions

### 1. Editorial

Good for:

- landing pages
- content-heavy surfaces
- product storytelling

Traits:

- strong typography
- generous whitespace
- restrained color
- fewer cards, more rhythm

### 2. Product UI

Good for:

- dashboards
- settings
- admin interfaces
- workflows

Traits:

- clearer grouping
- denser information
- stronger utility hierarchy
- controlled accents for actions and status

### 3. Premium Quiet

Good for:

- high-trust products
- portfolios
- polished branded experiences

Traits:

- fewer colors
- softer contrast
- refined spacing
- typography does more work than decoration

### 4. Playful But Controlled

Good for:

- consumer products
- lightweight tools
- onboarding and discovery

Traits:

- one bright accent family
- rounded forms used consistently
- selective motion
- friendly but still structured

## If The UI Looks Generic, Check These First

- Is there a clear type scale, or is everything near the same size?
- Is one accent color doing useful work, or is the palette muddy?
- Is spacing systematic, or just whatever looked okay locally?
- Is the background helping establish atmosphere, or is it a flat default
  surface?
- Are there too many generic cards with weak relationships between them?
- Would the layout still feel intentional without the framework defaults?

## Practical First Moves

If you do not know where to begin:

1. choose one of the four directions above
2. pick one accent color family only
3. define three text tiers:
    - hero / primary heading
    - section / feature heading
    - body / support text
4. define three spacing tiers:
    - tight
    - normal
    - spacious
5. keep one area visually loud and let the rest support it
